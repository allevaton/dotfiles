# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Personal fish shell configuration for a multi-platform setup (macOS primary, WSL, Linux). Uses Fisher for plugin management.

## Structure

- `config.fish` — main config: env vars, aliases, PATH, platform-specific setup
- `conf.d/` — plugin init scripts (auto-generated by Fisher, plus theme). Loaded alphabetically **before** `config.fish`
- `functions/` — lazy-loaded function files. Fish auto-loads from here on first call by matching filename to function name
- `completions/` — lazy-loaded completion files. Fish auto-loads on first tab-complete by matching filename to command name
- `secrets/` — cached secrets fetched from 1Password (gitignored, `chmod 600`). Not checked in
- `fish_plugins` — Fisher plugin manifest (source of truth for installed plugins)
- `fish_variables` — fish universal variables (auto-managed by fish, do not hand-edit)

## Plugins (managed by Fisher)

- **nvm.fish** — Node version management
- **fzf.fish** — fuzzy finder integration (Alt+C, Ctrl+T bindings)
- **z** — frecency-based directory jumping
- **catppuccin/fish** — color theme (Macchiato variant active)

## Secrets Pattern (IMPORTANT)

**Never hardcode secrets in config files, `fish_variables`, or environment variables.** Secrets are handled as follows:

1. Fetched from 1Password CLI (`op read`) on first use
2. Cached locally in `secrets/` with `chmod 600`
3. Injected via `env VAR=value command <tool> $argv` — scoped to the child process only, never exported to the shell environment

See `functions/jira.fish` for the reference implementation. Any new tool that needs a secret should follow this same pattern.

## Key Conventions

- Wrapper functions use `env VAR=value command <tool> $argv` to scope secrets to the child process — they never pollute the shell environment
- Plugin-managed files (conf.d, most of functions/, completions/) should not be hand-edited — Fisher overwrites them on update
- `conf.d/fish_frozen_theme.fish` is generated by `fish_config` — edit theme via that tool, not directly
- Platform detection uses `uname` for macOS and `WSL_DISTRO_NAME` for WSL
- External tool dependencies: bat, eza, starship, lazygit, moor, fzf, op (1Password CLI)
